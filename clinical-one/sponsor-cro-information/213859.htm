What if users have problems working in a live study?

Having problems trying to verify a study?

What if someone can't work in a live study?

  1. Ask a user administrator to confirm the user's effective date range in the study is not in the past or future.
  2. Contact Oracle Support.

What if someone opens a study, and it says they have no sites?

  1. Verify that you created sites for Production mode.
  2. Make sure the sites are all activated.
  3. Contact Oracle Support.

What if a site user can't add subjects in a study?

  1. If the site user has access to multiple sites, tell the user to make sure the site is selected from the Site drop-down, to the right of the Manage Subjects drop-down.
  2. Check with an administrator to make sure the user has a site user role for Production mode.
  3. Check whether the screening limits for the study have been met.
  4. Contact Oracle Support.

What if a site user can't randomize subjects or dispense kits in a study?

  1. Make sure the subject came in within the visit window, if the study doesn't allow dispensation out of window.
  2. Ask the site user to make sure that all required questions are answered.
  3. Ask a user administrator to verify that the site user is assigned to:
    • A site user role for Production mode.
    • The site that the user is trying to work with for Production mode.
  4. Make sure the study has:
    • A kit list for Production mode.
    • A randomization list for Production mode.
  5. Confirm that the randomization list is mapped to the study version that you making live.
  6. Check that kits:
    • Are assigned to the depots.
    • Have statuses of Available.
  7. Make sure the site has kits to dispense.
  8. Verify the following information for your manufacturing lots (and blinded lots, if you are using them):
    • Kits of every type are assigned to lots.
    • Expiration dates of all lots are in the future.
    • Expiration dates of all lots are late enough that kits can still be dispensed in consideration of the Do Not Dispense value for the visit.
  9. Check whether the screening limits and randomization limits for the study have been met.
  10. If you are randomizing into stratum groups, make sure the stratum groups don't have overlapping number ranges.
  11. Contact Oracle Support.

What if a depot can't raise shipments in a study, or a site can't request manual shipments?

  1. If the site can't request a manual shipment, check whether the study allows sites to request shipments.
  2. Check the depots:
    • Make sure a depot is assigned to send shipments to the country in which the site exists.
    • Check that kits are assigned to the depot.
  3. Make sure that kits that are assigned to the depot are also assigned to a manufacturing lot.
  4. Verify the following information for the manufacturing lots (and blinded lots, if you are using them):
    • Kits of every type are assigned to a lot.
    • Expiration dates of all lots are in the future.
    • Expiration dates of all lots are late enough that kits can still be dispensed in consideration of the Do Not Ship (DNS) Days value for the manufacturing lot.
  5. If your study requires label groups, make sure each kit is assigned to a label group.
  6. If the study allows single kit ordering, check whether the depot has the right combination of kits. If not, make kits available at the depot.
  7. Contact Oracle Support.

What if orders aren't being raised?

  1. Make sure the study has a kit list for Production mode.
  2. Check the depots:
    • Make sure a depot is assigned to send shipments to the country in which the site exists.
    • Check that depots are activated.
    • Check that kits are assigned to the depot.
  3. Verify the following information for the manufacturing lots (and blinded lots, if you are using them):
    • Expiration dates of all lots are in the future.
    • Expiration dates of all lots are late enough that kits can still be dispensed in consideration of the Do Not Ship (DNS) Days value for the manufacturing lot.
  4. Check that the site:
    • Has been activated.
    • Has a resupply strategy associated with it.
  5. Check the event that triggers the first shipment, either site activation or the first subject who starts a specified visit. A shipment is created only after the selected event occurs.
  6. Contact Oracle Support.

What if a site user can't see shipments that are In Transit?

  • Make sure the site user is assigned to the appropriate sites and depots.

Adding and editing users in a self-hosted environment

You create a user name for each user who logs in to Empirica Study.

When you create a new user, Empirica Study sends an email message to the user if you have provided an email address for the user. The message includes the new user name and password, and the URL from which to access Empirica Study. The From address is the address associated with the site option Feedback Email.

If you have enabled single sign-on, the Empirica Study user names that you create must be identical to their associated single sign-on user names in Oracle Access Manager. In addition, you must still provide a Empirica Study password. When single sign-on is enabled, Empirica Study user names and passwords are applicable if, for example, a Superuser logs in to Empirica Study natively using a remote desktop connection. You create Empirica Study passwords according to the password restrictions you specified on the Site Options page.

This procedure describes adding and editing Empirica Study users only. For information on creating single sign-on users in OHSIAMS Oracle Identity Self Service, refer to your OHSIAMS Oracle Identity Self Service documentation.

To add or edit a user:

1.         On the Users page, do one of the following:

  1. To add a user, click Add a New User.
  2. To edit a user, click next to the user that you want to edit, and then click Edit.

The Add/Edit User page appears.

2.         Enter the following:


Field

Description

Authentication

Indicates whether single sign-on is enabled for the user name.

  • SSO—Single sign-on authentication is enabled.
  • Local—Single sign-on authentication is disabled. The user is locally authenticated.

Username

Unique name (up to 100 characters) of the user account. You can also re-use deleted user names. This field appears only if you are creating a new user name. If you are using single sign-on, this user name must be identical to the associated single sign-on user name in OHSIAMS Oracle Identity Self Service.

First Name

The user first name.

Last Name

The user last name

Email

Optional e-mail address (or addresses, separated by commas) for the user. This is the default email sender address when the user sends feedback, and the default recipient address when the user requests email notification about run completion. For a user with appropriate permissions, it is also used if the user sends a message to all users, or receives notification of a user account lockout due to login failures.

Quota

Reserved for future use. Leave blank.

Login Group

Name of the login group to which the user belongs.

Password

(Local authentication only)

Password for the user, up to a maximum of 64 characters. Note that users can also modify their own Empirica Study passwords. Follow any recommendations by your organization related to creating secure passwords. Site options affect allowed passwords.

Confirm Password

(Local authentication only)

Re-enter the password.

3.         Select or deselect any of the following:


Field

Description

Superuser

Indicates whether the user is a Superuser. Available only if you are logged in as a Superuser.

  • If selected—The user is a Superuser.
  • If deselected—The user is not a Superuser.

Password never expires

(Local authentication only)

Indicates whether the user password never expires. At least one user should have a password that does not expire, and can access all users. Generally, this is user is the user with the Administer Users permission.

  • If selected—The user password never expires.
  • If deselected—The user password expires after the number of days specified in the Password Expiration site option.

User must change password at next login

Indicates whether the user must change his or her password during the next login.

  • If selected—A window appears during the login process, requiring the user to enter a new password. Empirica Study then deselects this option automatically when the user has reset his or her password.
  • If deselected—The user is not required to enter a new password.

Account locked

If selected, the user cannot log in until you clear the check box. This check box is selected automatically for a user who tries to log in with an incorrect password more than the number of times allowed by the Number of Attempts Allowed site option.

Account disabled

If selected, the user account is disabled. Disabled users can't log in or receive notifications. When a local user password has expired, the user account becomes disabled. To allow the user to log in again, you must assign a new password to the user and re-enable the user account.

4.         Click Save. When you have saved the user name, links to assign roles, assign permissions, change the user password, and rename the user appear on the page.

Note: If you do not click Save before continuing the next steps, the changes you made in previous steps are not saved.

5.         Optionally assign roles to the user.

6.         Optionally assign permissions to the user.

7.         Optionally change the user password.

8.         Optionally rename the user.

This link is available only if you are a Superuser or you have the Administer Users permission.

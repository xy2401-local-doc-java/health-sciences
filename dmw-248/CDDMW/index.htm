Let's build a study: Tasks you do simultaneously

Here’s what you can do at the same time in Central Designer, InForm, and DMW.

Central Designer

Create special InForm visits and forms

(Study Designer) Create the visits and forms that are required in InForm, and meet the special design requirements to make sure that the formdata is handled correctly in an InForm study.

Configure the query groups for the study

(Study Administrator) Create the query groups that you can add users to in InForm. Those users are identified as the users who can act on queries created by other members of the group in InForm.

Select the study's sponsor

(Study Administrator) Select from a list of possible study sponsors to associate your study with a specific sponsor, or add a new sponsor to the study.

Configure the system settings for the study

(Study Administrator) Select the InForm release you are deploying your study to, and configure the system configuration settings that you want to use for the study.

Validate your study

(Study Validator) Make sure that your study is well-formed before you send it to InForm.

InForm

Install and configure the skeletal study

(Installer) Before you can deploy a study, you must set up the basic framework to hold your study design and data. This framework is called a skeletal study.

Install the Central Designer certificate

(Installer) You must install a certificate on the InForm application server to verify that every deployment package you receive from the Central Designer server comes from a trusted source.

Install the Central Designer rule engine

(Installer) Run the InstallCentralDesignerFiles command to install and register the Central Designer rules engine, as well as other files the InForm application uses for automated deployments.

Create Deployment users

(Administrator) A deployment user is any sponsor user with the Study Deployment right. You create deployment users in the InForm user interface or with the MedML Installer utility.

DMW

Select internal InForm tables

(Administrator) Select tables and views to be available in all studies by default. An administrator performs this task one time, and the tables are selected for all studies.

Create the users required for InForm integration

(Administrator) For each InForm study database, create required Oracle and InForm users to enable InForm and DMW to exchange data.

Set up the Study File Watcher

(Administrator) Specify the file system location where the system will watch for data files to load into DMW for the study.

Create the study

(Study Builder) Specify a name and other attributes. Apply a study template (optional) to reuse clinical data models, transformations, and validation checks. You can delete any you don't need.

Create or reuse clinical data models for labs

(Study Builder) Create a set of tables to contain data from each lab to be used in the study.

Create or reuse target clinical data models for transformed data

(Study Builder) Add sets of tables to contain data formatted for review and analysis.

Create or reuse validation checks

(Study Builder) Create or copy complex edit checks on lab and clinical data.

Create public custom listings

(Study Builder) Custom listings make data review easier by limiting the columns and data displayed.

Create public filters

(Study Builder) Create complex filter groups for data reviewers to use and modify.

Let's build a study: Tasks you do sequentially

Here’s what you must do first in Central Designer, then InForm, then DMW.

Central Designer

Finalize your study design

(Various users) To finalize your study and make it ready for deployment to InForm, create the remaining objects that you need for your study, including but not limited to:

  • Visits
  • Forms and sections
  • Items
  • Rules
  • Data mappings
  • The study workflow

Validate your study

(Study Validator) Make sure that your study is well-formed before you send it to InForm.

Deploy your study

(Deployment User) Move the study from the design environment (Central Designer) to the production environment (InForm).

InForm

Collect data

(Site User) Collect subject data on forms.

DMW

Define remote locations

(Administrator) Define remote locations for the Dev, UAT, and Production InForm environments.

Define web service locations

(Administrator) Define web service locations for the Dev, UAT, and Production InForm environments.

Create an InForm clinical data model

(Study Builder) Set up the InForm connections and import metadata from InForm.

Load data from InForm

(Study Builder) Load data from the InForm Development database. Load data from the UAT and Prod database when ready.

Let's update a study

Here’s what you must do first Central Designer, then InForm, then DMW in order to update your study due to a change like a protocol edit.

Central Designer

Determine whether the required changes can be deployed as an in-place revision

(Study Designer) Certain changes to studies that have been deployed to a LIVE InForm instance are allowed to be made as in-place revisions, while others cause validation errors or warnings.

Update your study

(Study Designer) Update your study objects and make a study version change, or mark the changes as in-place revisions, if required.

Deploy your study

(Deployment User) Move the study from the design environment (Central Designer) to the production environment (InForm).

InForm

Collect data

(Site User) Collect subject data on forms.

DMW

Run the DMW metadata comparison report

(Study Builder) You can compare the metadata—table and column structure—in DMW and InForm for the same study. You can compare any lifecycle stage in DMW to any lifecycle stage in InForm.

Load InForm metadata

(Study Builder) Load the table and column structure of the InForm database for the study.

Migrating an existing installation to this release

Use this checklist to migrate an existing 1.3.8.1 or earlier InForm Adapter installation to InForm Adapter 6.2.

Steps to migrate to this release

Step

Task

For details, see...

1
  • If you are using Central Coding, stop the job scheduler.
  • If you are using CIS, stop the scheduled synchronization for the study.
  • If you are using ODM, stop any scheduled jobs that call the ODM Export interface.

Central Coding User Guide.

CIS Administration Guide.

2

Install InForm Adapter 6.2 on the InForm 6.2 server.

Installation overview.

3

Migrate the InForm study to InForm 6.2.

InForm Upgrade and Migration Guide.

4

Register the server adapter for the migrated study in the InForm Adapter database.

Register a server adapter.

5

Register the migrated study in the InForm Adapter database.

Register a study.

6

If you are using Central Coding:

  • Configure a certificate for InForm Adapter authentication.
  • Add the adapter for the migrated study.
  • Add the migrated study and associate it with the new adapter.

Central Coding Installation Guide.

Central Coding User Guide.

7

If you are using CIS, configure it to work with InForm Adapter 6.2:

  • Register the new InForm Adapter in CIS.
  • Remove synchronization connections with the old adapter and create synchronization connections with the new adapter.

CIS Administration Guide.

8

Migrate security settings from the old environment.

Migrate security settings from the old environment.

9

After you migrate all the studies to the new environment, decommission the studies from the old environment.

Decommission a study.

10
  • If you are using Central Coding, start the job scheduler.
  • If you are using ODM, start any scheduled jobs that call the ODM Export interface.

Configuring CIS behind a Proxy server

If your environment has the CIS software placed behind a Proxy server, you must update the following configuration files:

..\Clintrial Integration Solution\CentralAdmin\web.config

..\Clintrial Integration Solution\CISSync\bin\ PhaseForward.CISSynchAdapter.CISSyncProcessor.exe.config

To update the files:

  • Add the following attributes to the binding name="ICentralAdminService" and binding name="TransactionAdapterService":

    closeTimeout="00:01:00" openTimeout="00:10:00" receiveTimeout="00:10:00" sendTimeout="00:10:00" useDefaultWebProxy="false" allowCookies="false" proxyAddress= "<myproxy_url>"

    The content of both elements in those files should look like:

    <binding name="ICentralAdminService" maxBufferPoolSize="2147483647" maxReceivedMessageSize="2147483647" closeTimeout="00:01:00" openTimeout="00:10:00" receiveTimeout="00:10:00" sendTimeout="00:10:00" useDefaultWebProxy="false" allowCookies="false" proxyAddress="http://&lt;proxy_address">

    <binding name="TransactionAdapterService" maxBufferPoolSize="2147483647" maxReceivedMessageSize="2147483647" closeTimeout="00:01:00" openTimeout="00:10:00" receiveTimeout="00:10:00" sendTimeout="00:10:00" useDefaultWebProxy="false" allowCookies="false" proxyAddress="http://&lt;proxy_address">

If you do not update the CIS configuration files, you might experience connection problems. For example, the error "The underlying connection was closed: The remote name could not be resolved" might appear in the event log.

The error information includes the operation that was taking place at the point where the connection could not be acquired.
